Git入门篇

赵十八

2018-11-15

Git

开始

  • 首先我们需要去github官网申请git账号。git官网
  • 申请之后，我们进入自己的linux服务器
  • 然后在GitHub上加入这个公钥 配置公钥

  • 设置git命令的简写模式（alias）
  • 设置自己的名字和邮箱

开始

  • fork

如果你想使用别人的项目，就需要fork。

  • clone

进入一个你想存放项目的文件夹。

如图复制ssh地址。

然后就可以对代码进行修改和提交了。

  • checkout

项目有不同的分支。
一般本地主分支为master。
自己的远程分支为 origin/master
如果是合作项目，可能还会有一个公共仓库 xxx/master

可以用以下命令查看。

创建本地新分支: new_master 为你创建的新分支名，xxx/master为你要以哪条分支为基准创建new_master

将新分支推送至自己的远程分支。

  • commit

当我们修改某个文件，使用git st就可以看到哪些文件被修改了。
然后使用 git add 可以将我们修改过的文件添加进暂存区

commit为某次修改的描述，是阶段性的。
我们每完成一个功能，或者每修复一个bug，最好都进行一次提交。

最后将代码push到我们的远程分支

  • reset
    有时候我们会后悔添加了文件。那么可以用reset 返回

这样就可以返回add之前的文件状态。

同样的我们如果已经push到远程分支了

想要回到我们push之前的状态，或者再之前的某个版本。

以上两个步骤，第一步是查看我们这个分支的所有版本号。

复制你想要回退的版本号，然后执行第二步，就可以了。

  • fetch/merge

push之后我们的远程分支就会和本地分支的内容一样了。

但是如果我们是一个公共项目，那就需要并入公共仓库。

这个就需要管理员来操作了。

等管理员合并了之后，别人就需要fetch并且merge你的代码，以此来使大家的代码都是同步的。

每次push之前我们都应该先merge一下公共仓库的代码。
以免我们在旧代码上修改提交导致冲突。
